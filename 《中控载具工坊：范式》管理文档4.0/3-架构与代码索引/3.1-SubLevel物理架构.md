# SubLevel 物理架构

> 理解 SubLevel 是理解整个项目的基础。详细版见 `3.0/3-架构与文件结构/3.1-SubLevel物理架构.md`。
> 本文档仅列出核心概念和常用 API。

## 核心概念

SubLevel（子层级）：将一组方块从主世界"剥离"出来，打包成刚体，在独立物理空间中模拟运动，再通过坐标变换投射回主世界视野。

```
主世界 BlockPos/BlockState
  → 装配 (assemble)
  → SubLevel (6DOF 刚体)
  → 物理引擎更新 Pose3d
  → 渲染回主世界视野
```

## 常用 API

| 方法 | 说明 |
|------|------|
| `Sable.HELPER.getContaining(level, pos)` | 从世界坐标查找 SubLevel |
| `subLevel.logicalPose()` | 获取当前物理位姿 |
| `subLevel.getPlot().getLoadedChunks()` | 获取已加载 chunk（访问内部方块） |
| `subLevel.renderPose(partialTick)` | 客户端获取平滑插值的渲染位姿 |
| `RigidBodyHandle.of(subLevel)` | 获取刚体句柄（施加力/扭矩） |
| `Sable.HELPER.getVelocity(level, position)` | 获取速度 (m/s) |
| `subLevel.getMassTracker()` → `MassData.getMass()` | 获取精确总质量 |

## ⚠ 坐标系统：两种变换

这是最常见的踩坑点：

| 你要做什么 | 用哪种方式 |
|-----------|-----------|
| 找出 SubLevel 内有哪些方块 | `getLoadedChunks()` + chunk 偏移 |
| 确定 SubLevel 在世界空间中的物理位置 | `logicalPose().transformPosition()` |
| 获取渲染用平滑插值位置 | `clientSubLevel.renderPose(partialTick)` |
| 给 SubLevel 施加物理力 | `logicalPose().transformPosition()` 定位力施加点 |

## 约束系统参数

| 参数 | 含义 | 典型值 |
|------|------|:------:|
| Stiffness | 刚度（弹簧常数） | 1600 |
| Damping | 阻尼（抑制震荡） | 40 |
| Friction | 摩擦 | 0.3 |
