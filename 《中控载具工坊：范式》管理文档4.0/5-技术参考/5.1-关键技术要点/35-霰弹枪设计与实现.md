# 霰弹枪设计与实现

## 背景

霰弹枪（Shotgun）是炮塔系统的独立功能复刻。炮塔的避雷针枪管在 06-18 更换为末地烛（`Blocks.END_ROD`）后，避雷针作为炮管的全部代码被霰弹枪继承，形成独立的第二武器系统。

## 设计思路

霰弹枪 ≠ 炮塔子类或变体。设计采用**独立副本模式**（Copy+Pivot）：
- 不继承炮塔类，直接复制 `TurretBaseBlock`/`TurretBaseBlockEntity`/`TurretAimController` 并全局重命名
- 新 ComponentRoles：`SHOTGUN_BASE`、`SHOTGUN_YAW`、`SHOTGUN_PITCH`
- 新 AffiliationRoles：同名角色枚举
- 完全独立注册（ModBlocks/ModBlockEntityTypes/ModItems/ModCreativeModeTabs）

这样做的代价是代码冗余（~1300 行重复），但收益是：修改炮塔绝不会意外影响霰弹枪，反之亦然。两种武器可以独立调整参数、约束类型、物理行为。未来可以考虑提取公共武器底座抽象，但目前不需要。

## 文件结构

| 文件 | 说明 |
|------|------|
| `ShotgunBaseBlock.java` | 地毯形 KineticBlock + ICogWheel，setPlacedBy 自动装配 |
| `ShotgunBaseBlockEntity.java` | BE：~1300 行，装配/拆卸/约束管理/PD 伺服/静态注册表 |
| `ShotgunAimController.java` | `driveAnglesImmediate()` 单静态方法，消除 tick 边界等待 |

文件名不包含 "Turret" 以明确与炮塔的独立地位。

## 静态注册表

霰弹枪有自己的静态注册表（避免与炮塔的 `GRINDSTONE_OWNER_MAP` 冲突）：

| 注册表 | 作用 |
|--------|------|
| `SG_GRINDSTONE_OWNER_MAP` | 砂轮 UUID → 底座 BlockPos（供 GrindstoneConfigC2SPacket 查找） |
| `SG_ROD_OWNER_MAP` | 避雷针 UUID → 底座 BlockPos（同上） |
| `SG_GRINDSTONE_ANCHOR_MAP` | 砂轮 UUID → 锚点坐标（客户端配置界面） |
| `SG_GRINDSTONE_LINE_CACHE` | 砂轮 UUID → 轴线数据（AxisLineRenderer） |
| `SG_CARPET_LOCAL_POS_MAP` | 含地毯的 SubLevel UUID → 地毯局部坐标列表（三色焦点标记） |

## 霰弹枪特有的射击系统

### Box-Muller 正态散布

每次开火发射 8 根射线，用 Box-Muller 变换生成二维正态分布散布方向：

```java
double gauss1 = Math.sqrt(-2 * Math.log(rng.nextDouble()))
        * Math.cos(2 * Math.PI * rng.nextDouble());
double gauss2 = Math.sqrt(-2 * Math.log(rng.nextDouble()))
        * Math.cos(2 * Math.PI * rng.nextDouble());

double spreadDegH = clamp(gauss1 * SIGMA, -MAX, MAX);
double spreadDegV = clamp(gauss2 * SIGMA, -MAX, MAX);
```

| 参数 | 值 |
|------|----|
| PELLET_COUNT | 8 |
| SPREAD_SIGMA | 5.0° |
| SPREAD_CLAMP | ±15.0° |
| MAX_DISTANCE | 100 格 |
| MUZZLE_OFFSET | 0.6 格（向前） |

### 枪口偏移

炮塔使用 `origin` 不做偏移（炮口 = 射线起点），霰弹枪向前偏移 0.6 格：

```java
Vec3 muzzleOrigin = origin.add(dirNorm.scale(SHOTGUN_MUZZLE_OFFSET));
```

目的：霰弹枪的 8 射线散布可能导致部分弹丸射到枪塔自身 SubLevel，偏移 0.6 格确保所有弹丸起点在枪管前方，完全避免自伤。

### 冷却系统

霰弹枪独立 20 tick（1 秒）冷却，与炮塔 3 tick 冷却解耦：

```java
if (currentTick - lastShotgunFireTick < SHOTGUN_FIRE_COOLDOWN_TICKS) {
    return; // 冷却中
}
```

冷却在 `WeaponOverlay.fireFromRegistry()` 组级别检查。冷却就绪后，遍历所有已装配的霰弹枪同时开火。

## 音效系统

### 自定义 SoundEvent

```java
public static final Supplier<SoundEvent> SHOTGUN_FIRE = SOUND_EVENTS.register("shotgun_fire",
    () -> SoundEvent.createVariableRangeEvent(
            ResourceLocation.fromNamespaceAndPath(IACP.MODID, "shotgun_fire")));
```

### 客户端播放

使用 `playLocalSound()` 播放，音量 1.5×：

```java
mc.player.clientLevel.playLocalSound(
        turretPos.x(), turretPos.y(), turretPos.z(),
        ModSounds.SHOTGUN_FIRE.get(),
        SoundSource.PLAYERS,
        1.5f, 1.0f, false
);
```

### 音效文件

路径：`assets/iac_p/sounds/shotgun.ogg`

**已知问题**：当前文件实际为 MP3 重命名（扩展名改为 .ogg 但编码未转换），需手动用 Audacity 等工具导出为真实 OGG Vorbis 格式才能正常工作。

### 服务端清理

移除 `WeaponFireC2SPacket.handle()` 中的 `SoundEvents.EGG_THROW` 和 `SoundEvents.SNOWBALL_THROW` 播放。原因：霰弹枪 8 射线 × N 枪 × 2 音效 = 16N 个音效同时播放，产生严重噪音污染。霰弹枪开火音效仅在客户端播放一次（组级别一次）。

## 瞄准集成

霰弹枪与炮塔共享同一瞄准数据流（`TurretTargetC2SPacket`），数据包 handler 遍历时同时查找 `TURRET_BASE` 和 `SHOTGUN_BASE` 部件：

```
TurretTargetC2SPacket.handle()
  → 解析载具 UUID
  → turretEntries = ComponentRegistry.getComponents(uuid, TURRET_BASE)
  → shotgunEntries = ComponentRegistry.getComponents(uuid, SHOTGUN_BASE)
  → 合并瞄准两个群体
```

## 与炮塔的差异总结

| 特性 | 炮塔 (Turret) | 霰弹枪 (Shotgun) |
|------|:------:|:--------:|
| 枪管方块 | END_ROD（末地烛） | LIGHTNING_ROD（避雷针） |
| 射线数 | 1 | 8 |
| 散布 | 无 | σ=5°, ±15° 钳位 |
| 枪口偏移 | 0（使用渲染原点） | +0.6 格 |
| 冷却 | 3 tick | 20 tick |
| 音效 | 无自定义（使用原版投掷物音效） | 自定义 shotgun_fire.ogg |
| 注册表前缀 | TURRET_* | SHOTGUN_* |
