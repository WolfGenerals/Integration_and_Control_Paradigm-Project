# 《中控载具工坊：范式》管理文档 4.0

> 轻量版文档体系。3.0 版本（`../3.0/`）保留为历史存档供回溯。
> **代码行为是唯一真理，文档是地图不是副本。**

## 文档结构

| 路径 | 用途 |
|------|------|
| `README.md` | 本文件。**唯一**的文档管理规则 |
| `1-项目介绍/` | 新人入口：项目定位 + 编译指南 |
| `2-功能清单/` | 功能总表 + 各子系统说明 |
| `3-架构与代码索引/` | SubLevel 架构 + 自动生成的文件索引 |
| `4-当前状态/` | 活跃问题 + 已验证行为 |
| `5-技术参考/` | **一直有效的知识**：踩坑记录、技术解析、设计哲学 |
| `6-愿景/` | 项目愿景 |

## 核心规则（就 5 条）

1. **单一事实来源** — 每条信息只在一个地方维护，其他地方用链接引用
2. **不要重复 2.1** — 已完成功能的完整清单只在 `2-功能清单/2.1-功能总表.md` 里
3. **技术参考不动日志** — `5-技术参考/` 写结论和规则，不写过程；过程在 `3.0/7-开发日志/`
4. **变代码就变文档** — 改动功能行为时，顺手更新 `2-功能清单/` 对应描述
5. **新增文件就跑脚本** — 新增 Java 文件后运行 `工具脚本/scan_src.py` 更新文件索引

## 速查表

| 你想做什么 | 看哪个文件 |
|-----------|-----------|
| 了解项目功能 | `2-功能清单/2.1-功能总表.md` |
| 某个功能的详细设计 | `2-功能清单/` 对应子系统文档 |
| 理解 SubLevel 是什么 | `3-架构与代码索引/3.1-SubLevel物理架构.md` |
| 某个 Java 文件是干什么的 | `3-架构与代码索引/3.2-代码文件索引.md` |
| 当前有什么 Bug 或待办 | `4-当前状态/4.1-当前状态.md` |
| 遇到某个报错/症状 | `5-技术参考/5.2-踩坑快速索引.md` |
| 理解某个技术决策 | `5-技术参考/5.1-关键技术要点/` |
| 项目设计哲学 | `5-技术参考/5.4-设计哲学与路线图/` |
| 了解"为什么做了这个决定" | `3.0/7-开发日志/`（历史存档） |

## 文件索引维护

```bash
# 新增 Java 文件后运行
python 工具脚本/scan_src.py
```

脚本自动扫描 `src/main/java/` 生成索引骨架，人工补一句话职责即可。
