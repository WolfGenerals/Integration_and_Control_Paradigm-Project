# 项目定位

## 一句话

**中控载具工坊：范式 (Integration and Control : Paradigm, IAC-P)** 是 [Create Simulated](https://github.com/ryanhcode/sable) 模组套件的扩展，在 Minecraft 1.21.1 + NeoForge 中实现类似《Crossout》/《Besiege》风格的载具系统——让玩家建造的物理结构可以作为载具被驾驶和控制。

项目不涉及多人对战匹配、经济系统或游戏平衡，聚焦于**核心机制的验证**：让一个物理结构（SubLevel）能被玩家"上车"并以第三人称轨道摄像机操控。

## 我们相信什么（设计哲学）

### 物理不只是视觉，是可玩的

| 很多模组 | IAC-P |
|---------|------|
| 播放"悬挂压缩"动画 | 弹簧力真的在压缩，弹簧常量和阻尼值写在代码里 |
| 播放"轮子飞了"的爆炸粒子 | 轮子（未来）真的飞出去，刚体、碰撞箱、物理属性全都有 |
| 显示"速度：50 km/h"的 HUD | 那是 `linearVelocity.length() × 3.6` 算了给你的 |

### 部件比整体重要

大多数载具模组把一辆车做成一个整体实体。IAC-P 反过来：

- **每个方块都是一个独立部件**，有自己的物理属性
- **方块之间的连接是约束**，有强度、自由度
- **车辆不是"一个东西"**，是"一堆方块按约束组合在一起"

这意味着：你可以打掉任何一个方块，车辆性能取决于部件的**排列方式**，不仅仅是数据。

### 建造是游戏的一半

如果一辆车需要 2 小时设计、调试、试驾、改良——每一次战斗中的损伤都会有情感重量。**每辆车都应该有自己的"性格"**。

## 依赖关系

```
Minecraft 1.21.1
  └── NeoForge 21.1.230
       ├── Create 6.x
       │    └── Simulated-Project [1,2)
       │         ├── simulated    — 核心物理框架（SubLevel、装配、Sable 引擎）
       │         ├── aeronautics  — 航空模块（螺旋桨、热气球等）
       │         └── offroad      — 陆地模块（车轮、钻头等）
       └── IAC-P (本项目)
```

## 愿景速览

> 想象你走向自己用圆石、铁块和脚手架搭出来的战车，按下 F 键上车。发动机从怠速 800 RPM 开始轰鸣，你挂上 1 档，轮子转动。转弯时车身真实侧倾——不是动画，是物理引擎算出来的。你冲下山坡，四个轮子同时离地——你的战车真的在飞。
