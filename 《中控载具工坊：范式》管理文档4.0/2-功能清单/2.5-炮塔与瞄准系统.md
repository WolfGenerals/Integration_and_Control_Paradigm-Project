# 炮塔与瞄准系统

## 炮塔底座 (TurretBase)

- 地毯形方块（1/16 格），右键触发装配/拆卸
- 放置时自动装配砂轮（方向机）+ 避雷针（枪管）SubLevel
- 继承 `KineticBlock` + `ICogWheel`，提供 Create 齿轮系统 RPM 输入
- 已废弃抽象齿轮模式，使用位置模式 PD 伺服

## 约束系统

| 约束 | 连接 | 类型 |
|------|------|------|
| RotaryConstraint | 砂轮↔载具 | 方向机 Y 轴旋转 |
| SwivelBearing position-mode | 砂轮↔载具 | 位置模式 PD 伺服 |
| GenericConstraint | 避雷针↔砂轮 | 高低机 |
| FreeConstraint | 避雷针↔载具 | 碰撞禁用 |

约束坐标：使用底层绝对方块坐标 `getCenterBlock() + (0.5, 0.5, 0.5)`，不可用世界坐标或嵌入层相对坐标。

## 瞄准系统（06-15 重构终版）

**数据流**：

```
CameraMixin(每帧) → onClientTick(20Hz) → 计算炮塔→命中点方向
→ TurretTargetC2SPacket(hitX,hitY,hitZ) → 服务端
→ 变换到载具局部空间 → 每座炮塔计算 yaw/pitch → driveImmediate()
```

**关键设计**：
- **载具局部空间三维球坐标**：所有坐标变换到载具局部空间计算，天然消除载具偏航影响，载具翻转时角度正确
- **driveImmediate()**：设目标后立即执行 setMotor，消除 0~50ms tick 边界空等
- **ComponentRegistry O(1)**：用部件注册表替代 chunk 三重循环扫描炮塔底座
- **准星偏向锥角**：客户端弹道预测，±1° 锥角掩饰炮塔旋转延迟 + 网络 RTT

## 武器系统

- 1000 格射线检测，炮口伤害与摄像机瞄准分离
- 多炮台独立开火 (`fireAllTurrets` 遍历)
- 按住连发（isDown + 3 tick cooldown）
- 无视无敌帧（`entity.invulnerableTime = 0`）
- 每 2 tick 执行 `performRaycast` + `raycastCooldown` 降低开销

### 部件损坏系统（06-15 最终简化版）

**设计原则**：客户端射线检测（纯 Minecraft COLLIDER，无视 SubLevel 物理外壳线框）返回方块/实体碰撞箱表面的命中点，服务端直接用命中点坐标查找 SubLevel 并施加伤害。不做归属排除——所有 SubLevel 方块、所有实体一视同仁。

**数据流**：

```
客户端 fireAllTurrets()
  → renderPose(partialTick) 算炮口位置（无偏移，保持平滑）
  → raycastGeneric() 找命中点（纯 COLLIDER，穿透 SubLevel 线框）
  → activeFires.add(new TurretFireInstance(origin, hitPos))
  → 判断是否命中 SubLevel 方块
     → resolveSubLevelHit(hitPos) → 获取 SubLevel UUID + 局部坐标
     → 是 SubLevel → sendToServer(WeaponFireC2SPacket with localPos + UUID)
     → 非 SubLevel → sendToServer(WeaponFireC2SPacket with worldPos)

服务端 WeaponFireC2SPacket.handle()
  ├─ hasSubLevel=true → damageBlockAtLocal(level, UUID, localPos, damage)
  │    → 直接操作局部 BlockPos，跳过 world→local 变换
  │    → 用当前 pose 转世界坐标仅用于粒子效果
  │    → 解决"旋转体无敌"：目标移动/旋转后局部坐标不变
  └─ hasSubLevel=false → damageBlock(level, hitPos, damage)  // 传统路径
  ├─ 实体伤害: ProjectileUtil.getEntityHitResult() 射线-实体交点检测
  │    → 三层降级: ①bb.clip ②nearestSurfacePointOnAABB ③entityHit.getLocation
  │    → 确保粒子在碰撞箱表面而非实体脚下
```

**速度偏移补偿已移除**（06-15）：50ms/100ms 预测均取消，使用炮口原始坐标。客户端弹道不再跟随炮口移动——`TurretFireInstance.origin` 开火时固定。

**局部坐标修复**（06-16）：命中 SubLevel 方块时，客户端在开火瞬间计算 SubLevel 局部坐标和 UUID，服务端用当前 pose 转回世界坐标。消除目标快速移动/旋转导致的命中失效。

**自车防护**：仅保留 `rayTraceSubLevels` origin 边距检查（margin=1.0 格跳过炮管自身 SubLevel）。不再构建排除集，不再检查 SubLevelOwnership。自身载具部件也会受到伤害——这是简化后的预期行为，武器伤害对所有目标一致。

### 弹道渲染

**BulletTrailRenderer** 使用与 `AxisLineRenderer` 相同的已验证渲染模式绘制白色弹道线条：

- 渲染阶段：`AFTER_LEVEL`（与 `AxisLineRenderer` 一致，避免 `AFTER_PARTICLES` 下 `RenderType.LINES` 不可见的问题）
- 渲染类型：`RenderType.LINES`（标准 Minecraft 线条）
- 坐标约定：摄像机相对坐标（`world - camera`），`modelViewMatrix` 不含摄像机平移
- 缓冲刷新：`bufferSource.endLastBatch()`
- 炮口位置不再缓存：`TurretFireInstance` 仅存储 `subLevelId`（避雷针 UUID）和 `hitPos`，每帧通过 `renderPose(partialTick)` 动态计算炮口世界坐标（消除抖动）
- 降级方案：找不到 SubLevel 时在命中点画一个点（`drawPoint`）

**Veil 依赖**：
```gradle
compileOnly "foundry.veil:veil-neoforge-1.21.1:4.1.4"
```
仓库：BlameJared Maven（`https://maven.blamejared.com/`）
运行时由 bundled JAR 提供。

### 性能优化（06-15 终版）

- 热路径日志全部降级 `info→debug`（`PartDamageCache`/`WeaponFireC2SPacket`/`SableBlockHelper`）
- 客户端 `performRaycast` 从每 tick 改为每 2 tick（`raycastCooldown = 2`）
- `raycastGeneric` SubLevel 循环加入 AABB 快速剔除（减少不必要的局部空间 clip）
- 移除速度偏移补偿代码（50ms/100ms 预测均取消）
- **AffiliationRegistry 基础设施优化**：
  - 新增 `VEHICLE_TO_DIRECT_MEMBERS` 索引，`getOwnAffiliatedSet()` 从 O(n) 降为 O(1)
  - `WRITE_LOCK` 同步块保证原子注册/注销
  - `CAMERA_AIM` 策略提前返回，跳过 sameVehicle/sameFaction 检查
  - 性能监控：热路径纳秒计时，超 100μs 记录慢查询
- **WorldLoadHandler**：注册 `LevelEvent.Load`，世界加载时清空注册表防止数据残留

## 霰弹枪系统 (Shotgun)

霰弹枪是炮塔的独立功能复刻，使用 `Blocks.LIGHTNING_ROD`（避雷针）作为炮管（炮塔原本的避雷针已被替换为 `Blocks.END_ROD` 末地烛）。

### 设计定位

霰弹枪 ≠ 炮塔变体。它是炮塔模式的**独立副本**（不继承，直接复制并重命名）：
- 新文件：`content/blocks/shotgun/` 包下三个文件
- 新 ComponentRoles：`SHOTGUN_BASE`、`SHOTGUN_YAW`、`SHOTGUN_PITCH`
- 新 AffiliationRoles：同名角色
- 注册在 ModBlocks、ModBlockEntityTypes、ModItems、ModCreativeModeTabs

### 约束系统

与炮塔完全相同的约束链：

| 约束 | 连接 | 类型 |
|------|------|------|
| RotaryConstraint | 砂轮↔载具 | 方向机 Y 轴旋转 |
| GenericConstraint | 避雷针↔砂轮 | 高低机（ANGULAR_X 自由） |
| FreeConstraint | 避雷针↔载具 | 碰撞禁用 |

### 8 射线霰弹扩散

每次开火发射 8 根射线，使用 Box-Muller 变换生成高斯（正态）分布散布：

```java
double gauss1 = sqrt(-2 * log(rng)) * cos(2 * PI * rng);
double gauss2 = sqrt(-2 * log(rng)) * cos(2 * PI * rng);
double spreadDegH = clamp(gauss1 * SIGMA, -MAX, MAX);
double spreadDegV = clamp(gauss2 * SIGMA, -MAX, MAX);
```

| 参数 | 值 |
|------|----|
| 弹丸数（射线数） | 8 |
| 散布标准差 σ | 5° |
| 散布钳位 ± | 15° |
| 最大射程 | 100 格 |
| 枪口偏移 | 0.6 格（向前，防止自伤） |

### 开火冷却

- 霰弹枪独立冷却：20 tick（1 秒），与炮塔 3 tick 冷却解耦
- 冷却在组级别检查（`lastShotgunFireTick` 全局变量），组内所有霰弹枪同步冷却
- 冷却就绪后，组内所有已装配霰弹枪同时开火

### 自定义音效

- `ModSounds.java` 注册 `shotgun_fire` SoundEvent
- 客户端通过 `playLocalSound()` 播放，音量 1.5×
- 音效文件：`assets/iac_p/sounds/shotgun.ogg`
- **注意**：当前文件实际为 MP3 重命名，需转换为真实 OGG 格式才能工作

### 服务端音效清理

移除 `WeaponFireC2SPacket` 服务端 handler 中的 `SoundEvents.EGG_THROW` 和 `SoundEvents.SNOWBALL_THROW`（霰弹枪 8 射线 × 多枪 × 2 音效 = 16 个音效同时播放的噪音问题）。
