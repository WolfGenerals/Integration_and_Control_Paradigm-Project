# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in IAC-P, please **do NOT** open a public issue.

Instead, send a private report to the maintainer:

- **GitHub**: [@HaiNaBaiChuan75](https://github.com/HaiNaBaiChuan75) — open a private vulnerability report via the repository's "Security" tab
- **Email**: Available in the commit history

You should receive a response within **48 hours**. If you don't, please follow up.

## What to Include

- A clear description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Suggested fix (optional but appreciated)

## Scope

This security policy covers:
- The IAC-P mod source code
- The build process and CI/CD pipeline
- Any direct dependencies managed by this project

It does NOT cover:
- Dependencies (Create, NeoForge, Minecraft) — report those to their respective projects
- Modpacks that include IAC-P — report those to the modpack maintainer

## Policy

- Vulnerabilities will be acknowledged within 48 hours
- Fixes will be prioritized based on severity
- After a fix is released, the reporter may choose to disclose publicly
