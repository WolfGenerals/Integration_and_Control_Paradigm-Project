# 文档结构总览

> 本文档提供整个文档库的"导航地图"——每个文件夹和核心文件的用途、维护频率、当前健康状态。

---

## 文件夹总览

| 路径 | 类别 | 用途 | 维护频率 |
|:-----|:----|:-----|:---------|
| `0-文档管理指南/` | 元规则 | 🆕 文档更新规则 + 结构总览 + 开发工作流规范 + 文档体系评估 | 低（规则变更时） |
| `1-项目介绍/` | 入门 | 新人入口：项目定位、编译指南 | 很低 |
| `2-功能清单/` | 功能 | 功能总表 + 各子系统说明 | **中**（功能变更时） |
| `3-架构与文件结构/` | 架构 | SubLevel 架构、API 索引、文件索引 | 低（代码结构变更时） |
| `4-进度与规划/` | 进度 | **当前状态**：活跃问题、待办、已验证行为 | **中**（进度变更时） |
| `5-技术参考/` | **技术性资产** | **一直有效的知识**：踩坑、技术解析、设计哲学 | **高**（新踩坑/决策时） |
| `6-愿景/` | 愿景 | 项目愿景 | 很低 |
| `7-开发日志/` | **文学性记录** | **时间性的过程**：\"为什么\"、发生了什么 | 低（新周期追加） |

---

## 核心文件速查

| 需求场景 | 先看哪个文件 | 类别 |
|----------|-------------|:----:|
| 了解项目是做什么的 | `1-项目介绍/1.1-项目定位.md` | 入门 |
| 想编译运行项目 | `1-项目介绍/1.2-上手引导与编译.md` | 入门 |
| 项目有什么功能？状态如何？ | `2-功能清单/2.1-功能总表.md` | 功能 |
| 某个功能的详细设计 | 对应子系统的文档（如 `2.4-座舱动力系统/2. 发动机模型.md`） | 功能 |
| SubLevel 是什么？核心概念 | `3-架构与文件结构/3.1-SubLevel物理架构.md` | 架构 |
| 某 Java 文件是干什么的？ | `3-架构与文件结构/3.3-代码文件索引.md` | 架构 |
| 当前有什么 Bug 或待办？ | `4-进度与规划/4.1-当前状态.md §活跃问题 + §待办` | 进度 |
| 遇到了某个症状/报错 | `5-技术参考/5.2-踩坑快速索引.md` | **技术性** |
| 想理解某个技术决策的细节（一直有效） | `5-技术参考/5.1-关键技术要点/` + 对应编号 | **技术性** |
| 项目设计哲学和路线图 | `5-技术参考/5.4-设计哲学与路线图/` | **技术性** |
| 想了解开发过程/为什么做了某个决定 | `7-开发日志/` + 对应时间线 | **文学性** |
| 文档体系有什么缺口？改进方向？ | `0-文档管理指南/0.4-文档体系评估与改进建议.md` | **元规则** |
| 文档更新/新增前需遵循什么规则？ | `0-文档管理指南/0.1-文档更新规则.md` | **元规则** |
