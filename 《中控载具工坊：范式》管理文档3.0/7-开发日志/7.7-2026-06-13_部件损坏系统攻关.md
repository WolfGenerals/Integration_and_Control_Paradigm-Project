# 2026-06-13：部件损坏系统攻关

## 目标

实现 #1 优先级功能：**部件损坏系统**（Part Damage System）—— 武器子弹命中 SubLevel 物理结构的方块时，对方块造成伤害并最终摧毁，触发 Sable 物理约束自动断裂。

## 过程

### 第一轮：完整 HP 系统（06-12）
`PartData` + `PartDamageCache` + `SubLevelProjectileHandler`。`Sable.HELPER.getContaining(Vector3dc)` 返回 null → 无效果。

### 第二轮：简化裂纹系统（06-13 上午）
废弃 PartData，改用 `destroyBlockProgress`。3×3×3 邻域搜索。全部返回 null。

### 第三轮：SubLevelContainer 遍历（06-13 下午）
用 `SableCompanion.INSTANCE.getContaining(Level, BlockPos)` 替代 Vector3dc 查询。仍无效果。

### 第四轮：参考代码研究（06-13 下午）
研究 `tacz_aero_compat.SableBridge` + `AeronauticsHelper`，发现：
- **核心发现 1**：`getContaining(Level, Vec3i)` 查 plot grid（plot chunk 坐标约 2000 万），武器命中在物理世界坐标（约 500）→ 坐标空间不匹配
- **核心发现 2**：正确做法是 `getAllIntersecting(BoundingBox3dc)` 或 `SubLevelContainer` 遍历 + **pose 坐标变换**
- **核心发现 3**：SableBridge 的**完整射线追踪方案**——将整个射线变换到 SubLevel 局部空间后重新 Clip

### 第五轮：实现 SableBlockHelper 简化版（06-13 下午）
- 新建 `SableBlockHelper.java`（封装 SubLevel 查询工具）
- 重写 `PartDamageCache.damageBlock()`（精简为 5 击系统）
- 添加服务端全链路日志
- 添加客户端 P 键调试测试

### 第六轮：坐标方案迭代（06-13 下午→傍晚）
| 迭代 | 方案 | 结果 |
|------|------|------|
| ④ | `getContaining(Vec3i)` + 26 邻域 | ❌ 物理世界坐标不在 plot grid |
| ⑤ | `getAllIntersecting(点BB)` + pose 变换 | ❌ hitPos 不在物理 BB 内 |
| ⑥ | `SubLevelContainer` + 物理 BB + pose 变换 | ❌ 变换后局部坐标为空气 |

**第六轮的关键日志**：
```
hitPos=(529.55, 4.40, 309.71)
SubLevel 2be5c95d 物理 BB 包含该点 ✅
pose.transformPositionInverse(hitPos) → localBP=(20483080, 127, 20579335)
level.getBlockState(localBP) → AIR ❌
```
**最新根因**：武器 `raycastGeneric` 返回的 hitPos 是射线与 SubLevel AABB 表面的交点（`rayAABBIntersection` 计算），该表面点变换到局部空间后落在 SubLevel 边界外的空气上。
