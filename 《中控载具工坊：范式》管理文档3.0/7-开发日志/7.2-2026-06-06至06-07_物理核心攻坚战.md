## 已实现功能

### 1. 质量自适应扭矩（06-06）

**文件**：`CockpitBlockEntity.java`

```java
effectiveTorque = totalMass * 9.81 * TORQUE_WEIGHT_RATIO;  // 0.02
effectiveTorque = max(effectiveTorque, ENGINE_TORQUE * 0.5);
```

保证功率/重量比一致，重车扭矩自动增大。

### 2. 连续油门系统（06-06）

**文件**：`CockpitBlockEntity.java`

`throttleLevel` 0.0~1.0 连续值，`THROTTLE_RATE = 0.025/tick`（06-08 从 0.05 降低）。
- W → +0.025/tick，S → -0.04/tick（06-08 新增三段式：滑行衰减 0.015/tick）
- 松开 → 自动滑行衰减（06-08 新增 `THROTTLE_DECAY=0.015`）
- 扫描方式：CockpitBE.tick() 直接扫描 SubLevel 内悬挂方块

### 3. 扭矩平衡负载模型（06-06）

**文件**：`CockpitBlockEntity.java`

`loadFactor = Σ力需求 / Σ最大力` → `torqueRatio = 1 - loadFactor` → `netAccel = RPM_ACCEL × torqueRatio`
- < 0.3：引擎轻松
- > 1.0：过载拖转速（憋熄火感）

### 4. P 控制器增益提升（06-06）

`torqueGain * 0.02 * dt` → `torqueGain * 0.2 * dt`（10 倍），
新增 `maxWheelForce = torqueGain / radius` 扭矩上限截断。

### 5. 动态载荷转移（06-07）

**文件**：`SuspensionTestBlockEntity.java`

每 tick 跟踪 `prevLocalVelocity`，计算加速度，按轮位分配载荷调整：
- 参数：`LOAD_TRANSFER_SENSITIVITY=0.3`, `COG_HEIGHT=0.8`
- 效果：加速后轮增载、刹车前轮增载、转弯外侧增载
- 用户反馈：**"这是让载具活过来的关键"**

### 6. Brush 轮胎侧偏模型（06-07）

**文件**：`SuspensionTestBlockEntity.java`

替换纯阻尼侧向力：
- 侧偏角公式：`α = atan2(Vy, |Vx|)`
- Brush 模型：正弦峰值 ~9°，过峰值线性下降到 10% 残余
- `CORNERING_STIFFNESS=10.0`
- 防抖：侧偏角钳制 ±45°、totalSpeed > 1.0 阈值、1.0~2.0 平滑混合

### 7. 显示值/物理值分离（06-07）

**文件**：`SuspensionTestBlockEntity.java`

`frictionDemandRatio` 分母改用自然预算 `max(adjustedSpringImpulse, nm×dt×20.0)`，不再使用 500× 膨胀的 frictionBudget。

## 已修复 Bug

| Bug | 根因 | 修复 |
|-----|------|------|
| 转向时减速抖动 | Brush 模型中 tan(α) 在大侧偏角发散 | 侧偏角钳制 ±45° |
| 原地停放自旋 | forwardSpeedAbs > 0.5 阈值过低 | 改为 totalSpeed > 1.0 |
| 摩擦需求比显示永远 ~17% | 分母用 500× 膨胀的 frictionBudget | 改用自然预算 |
| 10kg 小车开不动 | TORQUE_WEIGHT_RATIO 使扭矩过小 | 调整 P 增益 + 确认 maxWheelForce |

## 活跃问题

1. **载荷转移侧向符号待验证**：右转时右轮应减载，当前公式可能相反
2. **SPRING_STIFFNESS_PER_NM=400（重车封顶 2000 N/m）偏软**：等效旧版 SUSPENSION_STRENGTH=5，重载支撑待测试
3. **P 控制器憋熄火感**：重车扭矩上限截断后 loadFactor 飙升，可能过于剧烈
4. **轻车（~10kg）仍不理想**：TORQUE_WEIGHT_RATIO=0.02 下扭矩仅 ~2 Nm
