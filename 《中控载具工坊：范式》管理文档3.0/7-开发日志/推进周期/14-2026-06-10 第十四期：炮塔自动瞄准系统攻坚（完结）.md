# 2026-06-10 第十四期：炮塔自动瞄准系统攻坚（完结）

### 概述
在炮塔底座物理约束链（砂轮+避雷针 SubLevel）装配成功的基础上，本期集中攻坚**炮塔自动瞄准系统**。经历了 3 个完整控制架构迭代+1 轮关键诊断后，最终实现了**稳定、无震荡、连续追踪**的炮塔瞄准。

### 关键里程碑

| 阶段 | 方案 | 结果 | 关键发现 |
|:----:|------|:----:|---------|
| **Phase I** 上午 | 基本武器系统 + 速度伺服 | ⚠️ 速度伺服到位不稳 | forward 方向 Bug + setMotor 不响应 |
| **Phase II** 下午 | 抽象齿轮模式（虚拟 RPM + 角度累积 + PD 伺服） | ❌ 目标附近震荡 | ~~猜想：kP 过刚/滑行距离~~ |
| **Phase III** 诊断 | DebugGear 实测 Create RPM 行为 | ✅ **RPM 为阶跃变化** | Create RPM 在 1 tick 内完成 0→±80 跳变 |
| **Phase IV** 最终 | **位置模式 PD 伺服（SwivelBearing 模式）** | ✅ **稳定无震荡** | 放弃 RPM 累积，直接 position-mode setMotor |

### Phase I：基本武器系统

#### 1. 射线检测系统 (WeaponOverlay)
- **WeaponOverlay.java**：从摄像机位置出发，最大 1000 格（后从 500 扩展到 1000），依次检测：
  - 传统 Minecraft 方块地形（ClipContext）
  - 实体（ProjectileUtil.getEntityHitResult）
  - SubLevel 物理结构 AABB（rayAABBIntersection Slab 算法）
- **关键过滤**：跳过当前乘坐的载具 SubLevel（`mountedUUID.equals()`），防止射线命中自身载具 AABB 表面
- **最小距离过滤**：排除 2.0 格内的命中
- **瞄空气**：无任何命中时返回射线末端点（类型 `"Air"`），使炮塔持续指向鼠标方向

#### 2. 准星与触发
- 上车时屏幕中央 2×2 白色像素准星
- B 键单发射线检测 + 发送 TurretTargetC2SPacket
- **持续自动瞄准**：每 client tick 自动执行 `WeaponOverlay.performRaycast()`（已取消旧 5 tick 冷却）
- Action bar 显示命中类型和距离

#### 3. 网络包系统
- `TurretTargetC2SPacket`：C→S，payload `(double targetX, double targetY, double targetZ)`
- 服务端：`TurretAimController.setTarget(gsUUID, x, y, z)` → `TARGETS` ConcurrentHashMap
- `PlayerMountTracker.tick()` 每 server tick 调用 `TurretAimController.tick(level)` 驱动

### Phase II：方向机控制迭代（~25 轮，最终废弃）

在 Phase I 武器系统基础上，方向机控制经历了约 25 轮迭代，以下是关键方案：

| # | 方案 | 结果 | 根因 |
|:-:|------|:----:|------|
| 1 | `applyTorqueImpulse` 扭矩脉冲 | ❌ 撕裂载具 | 早期约束未就绪 |
| 2-4 | `setMotor` PD 位置控制（SmartBlockEntity） | ❌ 转一次即停 | SmartBlockEntity 无 RPM 感知 |
| 5-7 | `addLinearAndAngularVelocity` 角速度 P 控制 | ❌ 不旋转 | forward 方向 Bug：(0,1,0)=UP |
| 8-10 | 速度伺服 + SNAP teleport | ❌ 到位不稳 | 恒速到死区后左右晃动 |
| **11+** | **抽象齿轮模式（TurretBase 改为 KineticBlockEntity）** | ❌ 震荡 | 见下方分析 |

#### 抽象齿轮模式设计（废弃）

**核心思想**：模拟真实齿轮箱永久啮合——齿轮永不断开，旋转时高 RPM + 高 kP 驱动，停止时 0 RPM 堵转制动。

**架构**：
```
Create 齿轮传动 (RPM) ──→ TurretBaseBE.tick()
     + 虚拟 RPM (瞄准)          ↓
                         角度累积 (targetAngleDegrees)
                               ↓
                         PD 伺服 (setMotor, 恒定 kP/kD)
                               ↓
                         RotaryConstraint 物理旋转
```

**关键变更**：
1. `TurretBaseBlock`：`Block` → `KineticBlock + ICogWheel`
2. `TurretBaseBlockEntity`：`SmartBlockEntity` → `KineticBlockEntity`
3. 方向取反：`dir = -(int)Math.signum(yawErr)`
4. 恒定 kP=2000/kD=160（移除动态 PD）

**效果**：方向正确，车体安全，但在目标附近左右摇摆无法稳定对准 ❌

### Phase III：关键诊断 —— DebugGear 与 RPM 阶跃

#### 调试齿轮 (DebugGearBlock)
为诊断震荡根因，创建了专用调试工具：
- 小齿轮方块，`RotatedPillarKineticBlock + ICogWheel`
- N 键切换调试输出（`DebugGearToggleC2SPacket` + `DebugGearBlockEntity.toggleDebug()`）
- 开启后每 tick 打印 `[DebugGear] tick=XXX speed=+XX.X angular=+X.XX°/t RPM=+XX`

#### 关键发现：Create RPM 是阶跃变化

实测数据（45 tick 窗口）：
```
tick=7509 speed=+0.0 angular=+0.00°/t RPM=+0     ← 启动前
tick=7510 speed=-16.0 angular=-2.29°/t RPM=-16    ← 第 1 tick 突然变 -16
tick=7526 speed=-88.0 angular=-12.60°/t RPM=-88   ← 第 17 tick 突然变 -88
tick=7554 speed=+0.0 angular=+0.00°/t RPM=+0      ← 下一个 tick 突然归零
```

**结论**：Create 应力网络的 RPM 变化是**瞬时的**（阶跃），不存在任何渐变速过程。这意味着"角度累积"的开环系统不可控——当 RPM 从 +80 突然跳到 0，累积角度瞬间被 PD 伺服追踪，但物理惯性导致过冲 → 震荡。

### Phase IV：最终方案 —— 位置模式 PD 伺服

#### 核心洞察

放弃"RPM → 角度累积"的开环思路，改用**位置模式 PD 伺服**（参考 Create 的 `SwivelBearingBlockEntity`）：

```diff
- 每 tick: RPM → 累积角度 → setMotor(位置模式)
+ 每 tick: setMotor(位置模式, 目标角度, kP, kD, false, 0)
```

#### 最终架构

```
TurretAimController (每 server tick)
    ↓  计算绝对目标角度
    ↓  targetYawRad = -(currentYawRad + yawErr) + offsetRad
    ↓  
TurretBaseBlockEntity.setTargetYawAbsolute(degrees)
    ↓  targetAngleDegrees = degrees（直接覆写，不累积）
    ↓
updateYawServo() 每 tick:
    └─ angleLerp(1.0, lastTarget, target) 做插值
    └─ setMotor(DEFAULT_AXIS, goal, kP=200, kD=16, false, 0)
    └─ 参考 SwivelBearingBlockEntity.updateServoCoefficients()
```

#### 关键代码

**TurretBaseBlockEntity.java** — 位置模式伺服：
```java
private void updateYawServo() {
    float goal = AngleHelper.rad(AngleHelper.angleLerp(1.0f,
            (float) lastTargetAngleDegrees, (float) targetAngleDegrees));
    swivelBearingHandle.setMotor(
            RotaryConstraintHandle.DEFAULT_AXIS,
            goal,           // 目标角度（弧度）
            SERVO_STIFFNESS, // kP = 200
            SERVO_DAMPING,   // kD = 16
            false,           // position mode
            0.0
    );
}

public void setTargetYawAbsolute(double degrees) {
    this.lastTargetAngleDegrees = this.targetAngleDegrees;
    this.targetAngleDegrees = degrees;  // 直接覆写，不累积
}
```

**TurretAimController.java** — 绝对角度计算：
```java
// 1. 计算当前朝向到目标的最短夹角
double yawErr = Math.atan2(tnx*fnz - tnz*fnx, clamp(fnx*tnx + fnz*tnz, -1, 1));

// 2. 从当前朝向 + 误差算出目标绝对角度
//    atan2 用逆时针为正，约束用顺时针为正 → 取反
double currentYawRad = Math.atan2(fnx, fnz);
double offsetRad = Math.toRadians(Config.TURRET_YAW_OFFSET.get());
double targetYawRad = -(currentYawRad + yawErr) + offsetRad;
tb.setTargetYawAbsolute(Math.toDegrees(targetYawRad));
```

#### 最终参数

| 参数 | 值 | 说明 |
|------|:---:|------|
| SERVO_STIFFNESS | 200.0 | PD 位置刚度（直接 kP） |
| SERVO_DAMPING | 16.0 | PD 速度阻尼（直接 kD） |
| YAW_MAX_STEP_DEG | 3.0°/tick | 比例增量限幅（约 60°/s） |
| YAW_GAIN | 80.0 | 比例增益 |
| YAW_DEAD | 0.5° | 死区 |
| PITCH_MAX_STEP_DEG | 2.0°/tick | 高低机增量限幅（约 40°/s） |
| PITCH_GAIN | 60.0 | 高低机增益 |
| PITCH_DEAD | 0.5° | 高低机死区 |
| PITCH_CLAMP | 60° | 俯仰最大角 |
| TURRET_YAW_OFFSET | 可配置（默认 0°） | 偏航校准偏移（Config.java） |

### 调试齿轮 (DebugGearBlock)

- **用途**：诊断 Create 应力网络的行为特征
- **类型**：`RotatedPillarKineticBlock + ICogWheel`，小型齿轮
- **触发**：N 键（`DEBUG_GEAR_KEY`），准星对准齿轮按 N
- **输出**：每 tick 打印 `[DebugGear] speed/angular/RPM`
- **关键诊断结论**：RPM 变化为阶跃，无渐变

### 可配置偏航校准

`Config.java` 新增 `turret.aim.yawOffset`：
- 类型：`ModConfigSpec.DoubleValue`
- 范围：-180° ~ 180°
- 默认：0.0°
- 用途：补偿炮塔瞄准位置与实际位置的固定角度偏差
