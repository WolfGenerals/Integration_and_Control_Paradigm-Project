## 写作规则

1. **聚焦于过程**：时间线、决策上下文、遇到的问题、最终结果
2. **不重复技术细节**：如果某个 Bug 的根因和修复方案在 `5-技术参考/5.1-关键技术要点/` 或 `5.2-踩坑快速索引.md` 中已有记录，日志中引用即可，不需要再写一遍
3. **文件变更清单**：列出新增/修改/删除的文件，方便追溯
