# 瞄准系统：角度直驱与视差修正

## 背景

炮塔瞄准经历了三次迭代：

| 迭代 | 方案 | 问题 |
|:----|:-----|:-----|
| v1（06-10） | 发命中坐标 → 服务端 atan2 转角度 → AimController TARGETS map → tick() 比例增量逼近 | server tick 边界等待 0~50ms；双重限速 |
| v2（06-14） | 发命中坐标 → ComponentRegistry 查找 → 直接计算角度设目标 | 省 tick 等待和 chunk 扫描，但仍需 atan2 坐标→角度 |
| v3（06-15） | 发摄像机朝向角度 → 服务端简单减法 | 最快路径，零额外运算 |

## 数据流

```
CameraMixin (每帧)     onClientTick (20Hz)      TurretTargetC2SPacket      服务端 packet handler
     │                       │                        │                          │
     │ 轨道球面坐标            │ cameraYaw/cameraPitch   │                          │
     │ lookDir + position ───→│ + 视差修正 ────────────→│ yaw/pitch ──────────────→│
     │                       │                        │  2 float                 │ turretYaw = yaw - vehicleYaw
     │                       │                        │                          │ turretPitch = -pitch
     │                       │                        │                          │ driveAnglesImmediate()
```

## 角度转换公式

### 偏航（Yaw）

```java
// 客户端：atan2 输出数学标准 (CCW+: 南=0°, 东=+90°)
// 取反转为 MC 惯例 (CW+: 南=0°, 西=+90°)
float targetYaw = (float) -Math.toDegrees(Math.atan2(tx, tz));

// 服务端：减去载具偏航（也是 MC CW+ 惯例，从四元数取反得到）
float turretYaw = targetYaw - vehicleYaw;  // 结果 = 炮塔相对载具的偏航
```

### 俯仰（Pitch）

```java
// 客户端：asin 在 MC 惯例下（负=上仰，正=下俯）
float targetPitch = (float) -Math.toDegrees(Math.asin(ty / dist));

// 服务端：取反转为炮塔惯例（正=上仰，负=下俯）
float turretPitch = -targetPitch;
```

### 载具偏航提取

```java
// 从 Sable SubLevel pose 四元数提取世界偏航
var q = new Quaterniond(vPose.orientation());
double yawRad = Math.atan2(2.0 * (q.w * q.y + q.x * q.z),
                           1.0 - 2.0 * (q.y * q.y + q.z * q.z));
float vehicleYaw = (float) -Math.toDegrees(yawRad);  // 取反：数学 CCW+ → MC CW+
```

## 视差修正（Parallax Correction）

### 问题

炮塔（C）和摄像机（A）位置不同。当两者朝向相同时，远程武器会因视差而偏离瞄准点。

```
    A (摄像机)     C (炮塔)
     \            /
      \          /
       \        /
        \      /
         O (命中点)
```

### 方案

客户端每 tick 计算炮塔→命中点方向的世界角度，直接作为修正后的目标角度发送，不经过命中坐标。

```java
// 炮塔→命中点方向向量
double tx = hitPos.x - turretPos.x();
double ty = hitPos.y - turretPos.y();
double tz = hitPos.z - turretPos.z();

// 炮塔应指向的世界角度（已含视差修正）
float yaw   = (float) -Math.toDegrees(Math.atan2(tx, tz));
float pitch = (float) -Math.toDegrees(Math.asin(ty / dist));

// 直接发角度，不发坐标
ModNetworking.sendToServer(new TurretTargetC2SPacket(yaw, pitch));
```

命中坐标仅在客户端本地用于角度计算，不通过网络传输，消除了坐标延迟抖动。

## 限速消除

`setTargetYawAbsolute()` 和 `setTargetPitchAbsolute()` 之前包含限速轨迹规划：

```java
// 旧：每 tick 最多移动 yawSpeedPerTick() 度
double step = Math.copySign(Math.min(Math.abs(delta), yawSpeedPerTick()), delta);
this.targetAngleDegrees += step;

// 新：直接覆写，PD 伺服自然平滑
this.targetAngleDegrees = degrees;
```

PD 位置模式 `setMotor` 的 angleLerp + Rapier 约束求解器本身提供平滑物理运动，软件层额外限速是冗余的。
