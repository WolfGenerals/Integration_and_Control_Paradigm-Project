# 十六、扭矩平衡负载模型

### 核心概念
转速变化率由扭矩余额决定，而非固定速率逼近目标。

```java
loadFactor = Σ(力需求) / Σ(引擎最大可输出力)

// 负载因子 < 0.3：引擎轻松
// 负载因子 0.3~1.0：有负担
// 负载因子 > 1.0：过载，转速被拖降

double torqueRatio = 1.0 - clamp(loadFactor, 0, 5);
double netAccel = RPM_ACCEL * torqueRatio;
```

### 负载因子计算（06-08 修正）

```java
// CockpitBE.calculateLoadFactor()
for each 悬挂方块:
    totalDemand += 该轮 P 控制器力需求 + 滚动阻力
    totalMaxForce += effectiveTorque × |ratio| × FINAL_DRIVE / wheelRadius  // ← 含主减速比

// ⚠ 分母必须除以轮数：累加 N 次后 totalMaxForce 是实际可用总力的 N 倍
return totalDemand / (totalMaxForce / count);
```

**06-08 修正**：原代码未除以 `count`，导致 loadFactor 被低估约 4 倍（4 轮车）。

### 物理直觉
- **5 档停车放手刹**：手刹拉着 → P 控制器不输出 → loadFactor ≈ 0 → 自由轰油
- **松手刹起步**：P 控制器巨大误差 → loadFactor > 1.0 → 转速暴跌 → 憋熄火感
- **高速巡航**：P 控制器误差小 → loadFactor < 0.3 → 高转速轻松维持
