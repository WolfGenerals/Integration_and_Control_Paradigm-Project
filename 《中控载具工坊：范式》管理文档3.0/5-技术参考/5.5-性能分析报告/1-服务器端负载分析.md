## 一级热点：Sable Rapier 物理引擎（~40%）

| 因素 | 对负载的影响 | 说明 |
|------|:----------:|------|
| SubLevel 数量 | **O(n)** | 每多一个 SubLevel，多处理一个刚体 |
| 每 SubLevel 碰撞形状复杂度 | **O(m)** | 方块数量决定碰撞网格顶点数 |
| 约束数量 | **O(c)** | RotaryConstraint、GenericConstraint 参与求解 |

### 典型负载对比
| 场景 | 物理负载 |
|------|:--------:|
| 空载（无 SubLevel） | 极低 |
| 行驶中 4 轮车 | **~100%（基准）** |
| 行驶中 4 轮车 + 炮塔 | **~140%** |
| 两辆 4 轮车同时行驶 | **~200%** |

## 二级热点：SuspensionBE.physicsTick()（~20% × 轮子数）

### 优化历史
| 日期 | 优化 | 节省 |
|:----:|------|:----:|
| 06-09 | CockpitBE 3→1 扫描合并 | 每 game tick 减少 2 次全量扫描 |
| 06-09 | SuspensionBE 缓存驾驶舱引用 | 每物理 tick 减少 2 次全量扫描 |
| 06-09 | VehicleDebugOverlay 移除回退质量扫描 | 消除极端情况下的额外扫描 |
| 06-09 | 移除 47 处 LOGGER.debug() | 消除字符串格式化和 I/O 开销 |

## 服务器负载与玩家人数的关系

```
负载 ∝ 活跃 SubLevel 数量 × 每 SubLevel 轮子数
```
玩家人数线性增长时，服务器负载线性增长——没有二次方效应。
