# 6. 资产策略：拒绝美术壁垒，社区驱动资产生态

> 决定日期：2026-06-07

## 核心原则

```
❌ 不投入精力/资金制作：
   ├─ 音效包（引擎声、开火、爆炸、环境）
   ├─ 视觉特效包（粒子、光效、屏幕震动）
   ├─ 武器纹理细节（枪口焰、弹道轨迹）
   └─ 自定义模型（除基础方块材质可随手画）

✅ 转而投入：
   ├─ 素材包导入系统（Resource Pack 式挂载）
   ├─ 素材包管理器（社区市场/评分/兼容性检查）
   └─ 开放的素材 API（让第三方开发者能写自定义特效）
```

## 为什么这样做

```
Crossout 模式                   中控载具工坊：范式
──────────────────────────────────────────────────
美术团队高薪烧钱              零美术预算
美术决定游戏上限              社区决定游戏上限
运营为回收美术成本不断逼氪     无运营压力
新玩家被付费墙劝退            无付费墙
代码维护被美术优先级挤占       全部精力投入玩法与物理
「你只能玩我们做好的」         「你想要什么，自己画」
```

## 社区资产生态设计（概念）

```
玩家创作
  ├─ 资源包（贴图/模型）
  ├─ 音效包（引擎/武器/环境）
  ├─ 特效包（粒子/光效/动画）
  └─ 完整主题包（以上全部 + 配置预设）
        ↓
  素材包管理器（游戏内 GUI）
  ├─ 浏览/搜索/排序
  ├─ 一键安装/切换
  ├─ 兼容性检查（版本/依赖）
  └─ 冲突检测
```

## 导入门槛设计

```
最低门槛：
  ┌─────────────────────────────────┐
  │  把 .png 扔进 resourcepacks/ 文件夹  │
  │  游戏内自动识别为可用素材包             │
  └─────────────────────────────────┘

进阶：
  ┌──────────────────────────────────────┐
  │  素材包 = JSON 清单 + 资源文件归档       │
  │  清单定义：名称/作者/版本/覆盖目标/兼容列表  │
  └──────────────────────────────────────┘
```
