## 首次构建流程

### 1. 提取嵌套 JAR

Create 6.x 和 aeronautics-bundled 使用 JarJar 将依赖打包在宿主 JAR 内部。NeoGradle 的 `implementation files()` **不会**自动解包嵌套 JAR，需要手动提取。

从 create JAR 提取 flywheel / ponder / Registrate：

```powershell
python -c "import zipfile; z=zipfile.ZipFile('libs/create-1.21.1-6.0.10.jar')
for n in ['META-INF/jarjar/flywheel-neoforge-1.21.1-1.0.6.jar','META-INF/jarjar/ponder-neoforge-1.0.82+mc1.21.1.jar','META-INF/jarjar/Registrate-MC1.21-1.3.0+67.jar']:
    with open('libs/'+n.split('/')[-1],'wb') as f: f.write(z.read(n))"
```

从 aeronautics-bundled 提取 offroad / simulated / aeronautics：

```powershell
python -c "import zipfile; z=zipfile.ZipFile('libs/create-aeronautics-bundled-1.21.1-1.2.1.jar')
for n in ['META-INF/jarjar/dev.ryanhcode.offroad.offroad-neoforge-1.21.1-1.2.1.jar','META-INF/jarjar/dev.simulated_team.simulated.simulated-neoforge-1.21.1-1.2.1.jar','META-INF/jarjar/dev.eriksonn.aeronautics.aeronautics-neoforge-1.21.1-1.2.1.jar']:
    with open('libs/'+n.split('/')[-1],'wb') as f: f.write(z.read(n))"
```

从 sable-neoforge 提取 companion：

```powershell
python -c "import zipfile; z=zipfile.ZipFile('libs/sable-neoforge-1.21.1-1.2.2.jar')
nested = z.read('META-INF/jarjar/sable-companion-common-1.21.1-1.6.0.jar')
with open('libs/sable-companion-1.6.0.jar','wb') as f: f.write(nested)"
```

提取后的 `libs/` 目录应包含：

```
libs/create-1.21.1-6.0.10.jar                  （原始 Create）
libs/create-aeronautics-bundled-...jar          （原始 bundled）
libs/flywheel-neoforge-1.21.1-1.0.6.jar        从 create 提取
libs/ponder-neoforge-1.0.82+mc1.21.1.jar       从 create 提取
libs/Registrate-MC1.21-1.3.0+67.jar            从 create 提取
libs/dev.ryanhcode.offroad...jar               从 aeronautics-bundled 提取
libs/dev.simulated_team.simulated...jar        从 aeronautics-bundled 提取
libs/dev.eriksonn.aeronautics...jar            从 aeronautics-bundled 提取
libs/sable-neoforge-1.21.1-1.2.2.jar           （原始 Sable）
libs/sable-companion-1.6.0.jar                 从 sable 提取
```

> **注意**：原始 JAR 仍需作为 runtime 依赖保留（NeoForge JarJar 机制需要）。提取的独立 JAR 仅用于编译期 classpath。

### 2. 编译

```bash
./gradlew build
```

### 3. 运行

```bash
./gradlew runClient
```

## 代码上手建议

1. **新协作者**：先读 `3-架构与文件结构/3.1-SubLevel物理架构.md` 理解 SubLevel 物理体系，再看 `2-功能清单/` 了解现有功能
2. **接需求**：在 `2-功能清单/` 确认需求，实现后更新对应文档
3. **修 Bug**：先查 `5-技术参考/5.2-踩坑快速索引.md` 看是否是已知问题，再参考 `3-架构与文件结构/3.3-代码文件索引.md` 定位文件
4. **避坑**：开发前翻阅 `5-技术参考/5.1-关键技术要点.md`
