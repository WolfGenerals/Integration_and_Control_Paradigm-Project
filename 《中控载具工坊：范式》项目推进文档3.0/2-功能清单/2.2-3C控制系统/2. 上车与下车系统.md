# 2. 上车/下车系统

### 上车流程

```
步骤1: 3 格射线检测
  └→ 获取命中方块位置

步骤2: 命中方块必须在 SubLevel 内
  └→ Sable.HELPER.getContaining(level, hitJomlPos)

步骤3: 检查驾驶舱核心
  ├→ 直接命中 Cockpit/Seat → 跳过扫描
  └→ 否则 → containsCockpit() 扫描 SubLevel chunk

步骤3.1: 结构完整性检查
  ├→ CockpitBlock → 检查上方 CockpitUpperBlock
  ├→ SeatBlock → 无需检查
  └→ 未命中驾驶舱 → findCockpitBlockInSubLevel() 查找

步骤3.2: 唯一性检查
  └→ 只能有一个驾驶舱组 + 一个核心下半截

步骤3.3: 独占检查
  └→ 同一 SubLevel 不能被两个玩家同时占用

步骤4: 执行上车
  ├→ PlayerMountTracker.mount()
  ├→ 禁用移动/碰撞
  ├→ 传送到 SubLevel 顶部
  └→ 发送 MountedStateS2CPacket 到客户端
```

### 上车条件（全部必须满足）

1. 射线命中某个在 **SubLevel 内**的方块
2. 该 SubLevel **包含一个驾驶舱核心**
3. 驾驶舱**结构完整**（下格上方必须有上格）
4. 该 SubLevel 内**仅有一个驾驶舱结构**
5. 该 SubLevel **未被其他玩家占用**

### 上车效果
- 玩家完全隐藏：`RenderPlayerEvent.Pre.setCanceled(true)`（模型/装备/名称全不渲染）
- 抑制粒子：`player.setInvisible(true)`（阻止受伤/进食/传送等粒子生成）
- 禁用移动：`flying = true`, `flyingSpeed = 0`, `noGravity = true`, 速度归零
- 无物理碰撞：`noPhysics = true`，服务端零碰撞箱
- 每 tick 将玩家位置同步到驾驶舱方块底部中心（经过 renderPose 变换）
- 客户端：强制第三人称背面，手部渲染取消（`RenderHandEvent`）

### 下车效果
- 恢复玩家状态：`noPhysics = false`, `flying = false`, `noGravity = false`
- 传送到安全位置（向后 2 格，若被阻挡则尝试周围 8 方向）
- 恢复第一人称

### 服务端每 tick
```java
for (每个已挂载玩家) {
    SubLevel 不存在？ → forceDismountClient()
    SubLevel 存在？ → 通过 logicalPose 将驾驶舱本地坐标变换到世界空间
    → setPos(transformResult)
    → 从 pose 旋转提取前进方向 → 设置 yBodyRot/yHeadRot
    → setDeltaMovement(Vec3.ZERO)
    → noPhysics = true, flying = true, noGravity = true
    → 设置零碰撞箱（当前位置）
}
```

### 断线重连
- 玩家重新进入世界时清除所有残留骑乘状态
- `EntityJoinLevelEvent` 清除 NBT 残留标记
- 服务端 `onServerTick()` 检测 player == null 自动清理

### SubLevel 占用追踪
```java
// SUBLEVEL_OCCUPANTS: SubLevel UUID → 玩家 UUID
mount()     → SUBLEVEL_OCCUPANTS.put(subLevelUUID, playerUUID)
unmount()   → SUBLEVEL_OCCUPANTS.remove(subLevelUUID)
断线/销毁  → 遍历中清理
```

清理入口有三个：主动下车、玩家断线、SubLevel 销毁。
