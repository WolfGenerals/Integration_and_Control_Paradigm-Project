# 6. WASD 智能映射（当前实现 + 进行中）

### 当前实现（已工作）

WASD 输入通过 `VehicleControlC2SPacket` 每 2 tick 发送到服务端，写入每个悬挂方块的 `applyControlInput()`：

| 输入 | 行为 |
|------|------|
| W | 全轮 `throttleForward = true` → CockpitBE 扫描到后 +throttleLevel |
| S | 全轮 `throttleBackward = true` → CockpitBE 扫描到后 -throttleLevel |
| A | 全轮 `targetSteeringYaw = +MAX_ANGLE` |
| D | 全轮 `targetSteeringYaw = -MAX_ANGLE` |
| 空格 | 全轮 `braking = true` |

**连续油门机制**（本次实现）：CockpitBE.tick() 通过 `scanThrottleDirection()` 扫描 SubLevel 内悬挂方块的 `isThrottleForward()/isThrottleBackward()` 状态，驱动 `throttleLevel`（0.0~1.0 连续值）渐进变化，而非直接跳变目标 RPM。

### 与 C 键配置的关系
WASD 映射作用于所有未经过 C 键 GUI 自定义的方块。若玩家通过 C 键将某个方块的某个按键修改为其他键，则该方块的对应动作使用自定义键。当前实现中，所有方块统一响应 WASD（因为 `VehicleControlC2SPacket` 携带的是标准按键状态，由客户端 `ClientEvents` 读取 WASD 后发送）。

### 待决策的设计点

- **与 C 键配置界面的关系**：WASD 映射是取代还是叠加？
- **多轮协调**：如何优雅地将单个输入分发到不同轮位（前轮转向、后轮驱动等）
