# 1. 架构总览与核心原则

## 架构总览

```
玩家输入 (W/S 油门) ──→ SuspensionBE.applyControlInput()
                           │
                           └─ throttleForward/Backward (布尔状态)
                                    │
                                    ▼
                         CockpitBE.tick() 每 tick 直接扫描 SubLevel
                         中所有悬挂方块，查询 isThrottleForward/Backward()
                                    │
                                    ├─ 连续油门调节 (0.0~1.0)
                                    ├─ 质量自适应扭矩计算
                                    ├─ 扭矩曲线 (RPM 依赖的扭矩修正)
                                    ├─ 负载因子计算 (loadFactor)
                                    ├─ 扭矩平衡转速更新
                                    │
                              变速箱 (gearRatio)
                                    │
                              差速分配 (/ N wheels)
                                    │
                               physicsTick 读取
                               PowertrainOutput
                               (wheelRpm, torqueGain)
```

### 核心原则
1. **油门状态通过座舱每 tick 直接扫描悬挂方块**获取，无共享 Map
2. **发动机转速在座舱的 `tick()` 中更新**，与游戏 tick 同步
3. **悬挂方块在 `physicsTick()` 中查询座舱获取动力输出**，直接作用于物理
4. **发动机-轮速耦合**（06-08 新增）：在档位中发动机转速不能低于 `轮速×齿比×主减速比`，模拟离合器结合的刚性传动
5. **换挡 RPM 同步**（06-08 新增）：换挡瞬间按齿比比例调整发动机转速，避免红线低扭
6. **指令转速分离**（06-08 新增）：P 控制器的目标速度使用油门指令转速（而非被耦合约束的实际转速）
7. **主减速比**（06-08 新增）：`FINAL_DRIVE_RATIO=14.0`，所有档位最终传动比 = 齿比 × 主减速比
