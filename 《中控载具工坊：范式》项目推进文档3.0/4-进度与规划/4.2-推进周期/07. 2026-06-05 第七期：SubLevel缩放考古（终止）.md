# 2026-06-05 第七期：SubLevel 缩放考古（终止）

### 背景
为实现 Crossout 的 3 倍精度网格匹配（比例耦合），需要在 Sable 的 SubLevel 上应用 1/3 均匀缩放。Java 侧 `Pose3d.scale` 字段早已存在且被渲染层使用，但 Rust Rapier 物理引擎从未接入该字段。

### 完成
- **上游代码考古**：确认 ryanhcode/sable 的 Rust 侧 `createSubLevel` JNI 只传 7 个 double（位置+四元数），无缩放
- **全链路修改方案论证并实现**（Rust JNI、Java JNI 桥接、粗筛 AABB、窄相碰撞、CI）
- **交叉编译 CI 全链路打通**（3 次失败后成功）
- **实际测试**：缩放参数成功写入引擎日志，但视觉和碰撞无实际变化

### 终止原因
- 缩放功能是上游**设计好 API（Java 侧）但物理引擎（Rust 侧）未实现**的烂尾部分
- 我们补全了 JNI 到窄相碰撞的链路，但实际效果不可感知
- 修复收益与投入不成正比，决定暂停

### 战略决策
- **放弃 Crossout 高精度路线（即等待缩放）**，转向 Blocky car 低精度路线
- 物理参数以 Minecraft 格子为基准，不依赖缩放
- 若未来缩放真的到来，蓝图导出时做后处理变换即可

详见 `7-历史记录/7.1-缩放考古记.md`。
