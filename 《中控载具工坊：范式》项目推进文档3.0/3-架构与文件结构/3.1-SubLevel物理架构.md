# SubLevel 物理架构

> 理解 SubLevel 是理解整个项目的基础。如果你只读一份文档，读这份。

---

## 核心概念

Minecraft 的方块世界本质上是静态的——每个方块固定在网格位置上。Create Simulated 通过 **SubLevel（子层级）** 机制，在不修改 Minecraft 方块体系的前提下，让一组方块获得 6 自由度刚体物理运动能力。

**基本原理**：将一组方块从主世界"剥离"出来，打包成一个刚体，在独立的物理空间中模拟运动，再通过坐标变换将渲染结果投射回主世界视野。

```
主世界 (BlockPos/BlockState)
  │  装配 (assemble)
  ▼
SubLevel (6DOF 刚体)
  │  物理引擎更新 Pose3d
  ▼
渲染回主世界视野
```

---

## 五大机制

### 1. SubLevel — 空间隔离

每个 SubLevel 拥有：
- **Plot**：一组方块的集合（局部坐标系）
- **Pose3d**：位置 + 四元数旋转，6 自由度位姿
- **物理属性**：质量、惯性张量、速度

```java
SubLevel subLevel = Sable.HELPER.getContaining(level, position);
Pose3dc pose = subLevel.logicalPose();
Vector3dc worldPos = pose.transformPosition(localPos); // 局部→世界
```

关键洞察：`logicalPose()` 返回 SubLevel 相对于主世界的变换矩阵。内部方块保持相对位置不变，整体平移旋转。

### 2. Assembly（装配）— 静态→动态

```
右键轴承 → assemble()
  → SimAssemblyHelper 扫描相连方块
  → 将方块从主世界转移到 SubLevel 内部存储
  → 创建刚体 + 添加约束
```

拆卸则反向操作，将方块还原到最近的主世界网格位置。

### 3. PhysicsPipeline — 独立物理时钟

物理 tick 独立于 Minecraft 的 20tps 游戏 tick，由 Sable 引擎驱动，使用可变时间步长：

```java
SableEventPlatform.INSTANCE.onPhysicsTick(MyClass::onPhysicsTick);
// timeStep 可变，精度不受游戏 tick 限制
```

### 4. 约束系统 — 连接 SubLevel 与世界

弹簧-阻尼器模型：

| 参数 | 含义 | 示例值 |
|------|------|--------|
| Stiffness | 刚度（弹簧常数） | 1600 |
| Damping | 阻尼（抑制震荡） | 40 |
| Friction | 摩擦 | 0.3 |

### 5. 渲染 — 投射回主世界

渲染时通过 `logicalPose()` 的变换矩阵将 SubLevel 内方块渲染到正确位置。支持 Flywheel 实例化渲染。

## 数据流

```
每物理 tick:
  PhysicsPipeline 计算力/扭矩
    → 更新 SubLevel.logicalPose (位置+旋转)
    → 回调 onPhysicsTick (磁铁/绳索/弹簧)

每渲染帧:
  SimpleSubLevelGroupRenderer 读取 Pose3d
    → 将 SubLevel 方块渲染到主世界坐标

玩家交互:
  F 键上车 → ServerMountHandler
    → Sable.HELPER.getContaining(level, hitPos) 查找 SubLevel
    → 检查驾驶舱 → 绑定玩家到 SubLevel
```

## 关键 API

| 方法 | 说明 |
|------|------|
| `Sable.HELPER.getContaining(level, pos)` | 从世界坐标查找 SubLevel |
| `subLevel.logicalPose()` | 获取当前物理位姿 |
| `subLevel.getPlot().getLoadedChunks()` | 获取已加载 chunk（访问内部方块） |
| `subLevel.getPlot().getCenterBlock()` | 获取中心方块的世界坐标 |
| `subLevel.getLevel()` | 获取主世界 Level 实例 |
| `subLevel.renderPose(partialTick)` | **客户端**获取平滑插值的渲染位姿 |
| `RigidBodyHandle.of(subLevel)` | 获取刚体句柄（施加力/扭矩） |
| `ForceTotal.applyImpulseAtPoint()` | 在指定点施加冲量 |
| `Sable.HELPER.getVelocity(level, position)` | 获取速度（返回 m/s） |
| `subLevel.getMassTracker()` → `MassData.getMass()` | 获取精确总质量 |

## ⚠ 坐标系统：两种不同的坐标变换

这是最常见的踩坑点。SubLevel 涉及两套坐标变换逻辑：

### 方块访问 → chunk 迭代
```java
// ✅ 正确：遍历已加载 chunk
for (PlotChunkHolder chunk : plot.getLoadedChunks()) {
    int chunkMinX = chunk.getPos().getMinBlockX();
    int chunkMinZ = chunk.getPos().getMinBlockZ();
    BlockPos worldPos = new BlockPos(x + chunkMinX, y, z + chunkMinZ);
    BlockState state = level.getBlockState(worldPos); // ✅ 正确的方块
}
```

### 空间定位 → logicalPose
```java
// ✅ 正确：计算座位高度
Pose3dc pose = subLevel.logicalPose();
for (int[] corner : corners) {
    pose.transformPosition(new Vector3d(x, y, z), temp);
    worldMinY = Math.min(worldMinY, temp.y); // ✅ 正确的世界空间位置
}
```

### 决策表

| 你想要做什么 | 用哪种方式 |
|-------------|-----------|
| 找出 SubLevel 内有哪些方块（内容扫描） | `getLoadedChunks()` + chunk 偏移 |
| 确定 SubLevel 在世界空间中的物理位置 | `logicalPose().transformPosition()` |
| 获取渲染用的平滑插值位置 | `clientSubLevel.renderPose(partialTick)` |
| 给 SubLevel 施加物理力 | `logicalPose().transformPosition()` 定位力施加点 |

核心区别：
- **内容访问**（"里面有什么方块"）→ chunk 迭代，因为方块存储坐标不同于物理位姿
- **空间定位**（"它在世界的哪个位置"）→ `logicalPose()`，物理引擎通过它对 SubLevel 做整体平移/旋转

## SubLevel 公共接口 vs Companion 接口

`SubLevel` 的公共接口方法有限。以下方法**仅在 companion 模块**中定义：

| 方法 | 定义位置 |
|------|---------|
| `logicalPose()` | `SubLevelAccess`（companion） |
| `getPlot()` | `SubLevelAccess`（companion） |
| `getLevel()` | `SubLevelAccess`（companion） |

**校验方式**：编译时若报 `无法访问SubLevelAccess`，说明 companion JAR 未加入 classpath（见 `1-项目介绍/1.2-上手引导与编译.md` 的提取步骤）。
