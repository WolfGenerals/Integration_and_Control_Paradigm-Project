## 06-08：调参优化攻坚战

### 动机

06-06/06-07 的动力系统重构引入了质量自适应扭矩、连续油门、扭矩曲线等机制，但存在若干缺陷：
- 极速崩盘（扭矩曲线平坦化不够 + RPM耦合双重打击）
- 无速度阻尼导致无限加速
- 载荷转移侧向符号反了
- 无自动衰减导致松油门不减速

### 完成列表

#### 扭矩曲线
**文件**：`CockpitBlockEntity.java`

```java
double rpmRatio = (engineRpm - ENGINE_IDLE_RPM) / (ENGINE_MAX_RPM - ENGINE_IDLE_RPM);
double shapedRatio = pow(rpmRatio, TORQUE_CURVE_SHARPNESS);  // 1.0
double curveMultiplier = TORQUE_IDLE_FRACTION + (1.0 - TORQUE_IDLE_FRACTION) * sin(π * shapedRatio);
this.effectiveTorque *= curveMultiplier;
```

| 参数 | 旧值 | 新值 | 效果 |
|------|:----:|:----:|------|
| `TORQUE_IDLE_FRACTION` | 0.35 | **0.80** | 平坦化，极速不再崩溃 |
| `TORQUE_CURVE_SHARPNESS` | — | 1.0 | 标准正弦 |

#### 发动机-轮速耦合
**文件**：`CockpitBlockEntity.java`

```java
// 每 tick 更新 engineRPM = max(轮端等效RPM, 怠速)
double wheelRPM = avgWheelAngularVel * 60.0 / (2π);
this.engineRpm = max(wheelRPM * gearRatio * FINAL_DRIVE_RATIO, ENGINE_IDLE_RPM);
```

- 换挡瞬间 RPM 同步：`engineRPM *= newRatio / oldRatio`
- P 控制器目标用指令转速而非被耦合的实际转速（分离策略）

#### 主减速比 FINAL_DRIVE_RATIO = 14.0
替代之前"改齿比压极速"的思路。直接档（5档，ratio=1.0）下轮端 RPM = 发动机 RPM / 14。

#### 阻力系统
**文件**：`SuspensionTestBlockEntity.java`

- `DRAG_COEFFICIENT = 0.0045`：二次方速度阻尼，自然限制极速 ~120 km/h
- 滚动阻力清理：移除 `/0.4` 向后兼容残骸，`crrBase 0.015 → 0.035`

#### 参数联动调整

| 参数 | 旧值 | 新值 | 原因 |
|------|:----:|:----:|------|
| `MIN_IMPULSE_MULTIPLIER` | 500 | **30** | 摩擦预算从 45g 降到 2.75g |
| `CORNERING_STIFFNESS` | 10 | **20** | 补偿 MIN_IMPULSE 降低后的回正刚度 |
| `THROTTLE_DECAY` | — | **0.015** | 新增：滑行自动降油 |
| `THROTTLE_BRAKE_RATE` | — | **0.04** | 新增：S 键主动减油 |

#### 载荷转移侧向符号修复
**文件**：`SuspensionTestBlockEntity.java`

```java
// 旧（错误）
double latTransfer = LOAD_TRANSFER_SENSITIVITY * mass * -accelX * COG_HEIGHT / trackWidth;  // ❌
// 新（正确）
double latTransfer = LOAD_TRANSFER_SENSITIVITY * mass * +accelX * COG_HEIGHT / trackWidth;  // ✅
```

`+accelX` = 向右加速，右转时右轮正确减载。

#### loadFactor 分母修正
```java
// 旧（错误）
double totalMaxForce = sum(maxWheelForce);  // 每轮上限各自累加 → 低估 4×
// 新（正确）
double totalMaxForce = sum(maxWheelForce);
totalMaxForce /= count;  // 平均值作为分母
```

### 踩坑

| 问题 | 根因 | 诊断方法 | 修复 |
|------|------|---------|------|
| 极速从 80 崩到 20 km/h | TORQUE_IDLE_FRACTION=0.35 平坦化不够 + RPM耦合双重打击 | 打印 engineRPM 与有效扭矩 | 0.35→0.80 |
| 无限加速 | 无速度阻尼 | 极速测试发现持续加速 | DRAG_COEFFICIENT=0.0045 |
| 右转右轮不减载 | 侧向载荷公式符号问题 | 打印载荷分布 | `-accelX` → `+accelX` |
| 松油门仍然加速 | 无自动衰减 | 用户反馈 | THROTTLE_DECAY=0.015 |

### 教训
1. **扭矩曲线 + RPM 耦合是双重非线性**：两个非线性系统叠加后行为不可预测，需逐项隔离测试
2. **参数联动**：任何核心参数调整后需全面回归测试关联参数
3. **速度阻尼是速度型载具的必需品**：无阻尼时物理引擎无法自然限制极速

## 06-09：炮塔系统攻坚战（~15 轮迭代）

### 动机
将避雷针作为炮管绑定到砂轮炮塔底座上，实现方向机（Y 轴旋转）+ 高低机（X 轴俯仰）。

### 目标约束链
```
载具 ── RotaryConstraint(Y) ── 砂轮 ── GenericConstraint(ANGULAR_X 自由) ── 避雷针
```

### 迭代史（15 轮）

| 轮次 | 方案 | 结果 | 教训 |
|:----:|------|:----:|------|
| 1 | 避雷针偏移 Y+1 | ❌ 消失 | 撞入载具方块 |
| 2 | FixedConstraint 同位置 | ❌ 巨大下拉力 | FixedJointBuilder 不更新 rotation |
| 3 | RotaryConstraint 同位置 | ❌ 飞走 | RevoluteJointBuilder 锚点传错 |
| 4 | FreeConstraint 同位置 | ❌ 重力坠落 | 6 DOF 全自由 |
| 5 | FixedConstraint 偏移 Y+1 | ❌ 撕裂砂轮 | FixedJointBuilder 不稳定 |
| 6 | GenericConstraint 偏移 | ❌ 消失 | 额外 FreeConstraint 干扰 |
| 7 | **GenericConstraint 同位置** | ✅ **稳定** | 去除跨级约束 + 同一姿态 |
| 8 | **解锁 ANGULAR_X** | ✅ **俯仰成功** | GenericJointBuilder 正确更新旋转帧 |

### 核心发现

#### 1️⃣ 两个 SubLevel 完全重叠可行
```
之前: 认为"两个刚体在同一位置 = 数值奇点"    ❌
现在: 完全同一位置 + setContactsEnabled(false)  ✅
```
Rust 侧 `joints.rs tick()` 计算 `local_anchor = pos_a - center_of_mass`。单方块 SubLevel 的 `pos_a ≈ center_of_mass ≈ getCenterBlock()+0.5` → `local_anchor = (0,0,0)` → 零静态力。

#### 2️⃣ 所有 SubLevel 必须完全一致的 pose
**position 和 orientation 都必须相同**。约束系统的 pos/normal 通过各 SubLevel 的 `logicalPose()` 变换到世界空间，姿态不同则 normal 指向不同 → 约束自相矛盾 → 撕裂载具。

#### 3️⃣ 不需要跨级加约束
```
之前: 避雷针↔载具 FreeConstraint（防碰撞）    ❌ 多余
现在: 避雷针只连砂轮，约束链自然传递         ✅
```

#### 4️⃣ GenericConstraint 比 FixedConstraint 更稳定
| 约束类型 | Rust builder | rotation_a | tick 旋转更新 | 稳定性 |
|---------|-------------|:----------:|:------------:|:------:|
| `FixedConstraint` | `FixedJointBuilder` | `None` | ❌ | ❌ 飞走 |
| `GenericConstraint` | `GenericJointBuilder` | `Some(identity)` | ✅ | ✅ |

### 俯仰解锁

```java
// 全锁定 → 保留 LINEAR_XYZ + ANGULAR_YZ，解锁 ANGULAR_X（俯仰）
GenericConstraintConfiguration bindConfig = new GenericConstraintConfiguration(
    pos1, pos2,
    new Quaterniond(), new Quaterniond(),
    EnumSet.of(
        ConstraintJointAxis.LINEAR_X,
        ConstraintJointAxis.LINEAR_Y,
        ConstraintJointAxis.LINEAR_Z,
        ConstraintJointAxis.ANGULAR_Y,
        ConstraintJointAxis.ANGULAR_Z
    ) // ANGULAR_X 未锁定 = 可俯仰
);
```
