## 一、WASD 智能映射系统

### 核心组件

| 组件 | 说明 |
|------|------|
| `VehicleOrientationData` | Record 记录 SubLevel 悬挂 HORIZONTAL_FACING 分布 (N/S/E/W) |
| `ClientMountHandler.scanOrientation()` | 遍历 SubLevel 所有悬挂方块，统计朝向并按 UUID 缓存 |
| `VehicleOrientationScreen` | C 键交互式 GUI：FACING 统计 + 3 按钮（Car Mode/Reverse/Toggle） |
| `SmartMapC2SPacket` | C→S 数据包，Action 枚举：CAR_MODE / REVERSE / TOGGLE_SMART |
| `SmartMapC2SPacket.handleCarMode()` | FACING 投票→宽度轴→质心 Z→智能按键分配（南北半区转向镜像） |
| `SuspensionTestBE.smartKey*` 字段 | 智能按键存储 + `getActiveKey*()` 自动回退到手动键 |
| `CockpitBE.smartMappingActive/Reversed` | NBT 持久化的智能映射开关/反转标志 |
| `CockpitBE.getWheelOutput()` | 反转时齿比符号反转 |

### 关键技术决策

**Car Mode 算法**：
1. 统计 N/S/E/W 四个方向的悬挂方块数量
2. 找到最多朝向的那个方向 → 前进轴
3. 前进轴的法线方向 → 宽度轴
4. 计算质心 Z（所有悬挂方块在宽度轴上的投影中位数）
5. 质心 Z 两侧各分配转向键（左半区→A=左转，右半区→D=右转，南北半区转向镜像）

**Reverse 反转**：
- 客户端：`localSwapSmartKeys()` 在发送包后立即交换本地智能键
- 服务端：`smartMappingReversed` 标志持久化到 NBT
- 引擎层：`tick()` 反转方向解读，`getWheelOutput()` 反转齿比符号

### 反转方向 Bug 修复

**症状**：按下 Reverse 后方向盘反转，但车辆不响应（油门不上）

**根因**：两层问题叠加
1. 按键层 W↔S 交换后，`throttleBackward=true` 被引擎层解读为 `direction<0` → 松油门
2. 齿比符号未反转，轮子方向与油门方向不匹配

**修复方案**：
- CockpitBE 新增 `smartMappingReversed` 字段（NBT 持久化）
- `tick()` 中：`direction = reversed ? -rawDir : rawDir`（反转方向解读）
- `getWheelOutput()` 中：`ratio = reversed ? -currentRatio : currentRatio`（反转齿比符号）
- 客户端：`localSwapSmartKeys()` 消除服务端同步延迟

## 二、上下车系统全面优化

### 优化背景

原有上下车系统存在 3 个问题：
1. **重复的 SubLevel chunk 扫描**：mount 过程中 `containsCockpit`→`hasUniqueCockpit`→`findCockpitBlockInSubLevel` 三次独立扫描
2. **客户端每 2 tick 全量扫描**：`sendVehicleControlInput()` 每 2 tick 遍历 SubLevel 所有 chunks 找悬挂方块
3. **下车位置简单**：只在 ±1 格范围内找空气，没有考虑危险方块和物理结构

### SubLevelScanner 工具类

**`events/SubLevelScanner.java`** 将常见的三重嵌套循环（chunk 遍历→本地坐标→世界坐标）提取为统一工具：

```java
public static void forEachBlock(SubLevel, Level, BlockVisitor);
public static void forEachBlockState(SubLevel, Level, BiConsumer<BlockPos, BlockState>);
```

替换了 **9 处**独立 chunk 扫描：
- `PlayerMountTracker`: findCockpitGroups/containsCockpit/hasUniqueCockpit
- `ServerMountHandler`: findCockpitBlockInSubLevel/resetSuspensionInputsInSubLevel
- `ClientMountHandler`: scanOrientation/syncSmartMappingState/localSwapSmartKeys
- `ClientEvents`: sendVehicleControlInput（改用缓存）
- `CockpitBlockEntity`: scanSubLevel（每 tick 执行的关键路径）
- `SuspensionTestBlockEntity`: invalidateCachesInSubLevel/findCockpitInSubLevel/countSuspensionBlocksInSubLevel

> ⚠️ **踩坑**：Java lambda 中不能修改局部变量（effectively final 限制），需要使用单元素数组 `int[] count={0}` 绕过。

### 合并 mount 时 chunk 扫描

在新方法 `scanSubLevelForCockpit()` 中一次遍历完成全部判断：

```java
// 替代三次扫描
CockpitScanResult result = PlayerMountTracker.scanSubLevelForCockpit(subLevel, level);
result.hasCockpit()   // ← 替代 containsCockpit
result.isUnique()     // ← 替代 hasUniqueCockpit
result.cockpitWorldPos() // ← 替代 findCockpitBlockInSubLevel
```

### 客户端悬挂位置缓存

- `ClientMountHandler.SUSPENSION_POSITIONS` 上车时一次性填充
- `sendVehicleControlInput()` 直接遍历缓存列表
- 每 2 tick 完整 chunk 扫描 → O(n) 列表遍历

### 双模式下车逻辑

| 条件 | 下车位置 | 搜索算法 |
|:----:|---------|---------|
| 按手刹+F | 附近地面 | 半径 1~5 格 8 方向，向下找固体地面，检查脚+头=空气，无危险方块，不与 SubLevel 物理结构重叠 |
| 直接 F | 载具顶部 | 扫描最高 Y，站在顶部+1，必要时偏移 |

**危险方块黑名单**：LAVA、FIRE、MAGMA_BLOCK、CACTUS、CAMPFIRE、NETHER_PORTAL、END_PORTAL、POWDER_SNOW

### 断线重连加固

- `PlayerMountTracker.onServerStarting()`：服务端启动时清空静态 Map
- `ClientMountHandler.onClientLogin()`：强制清除所有缓存
- 服务端 `onServerTick()` 检测 `player == null` 自动清理

### 已知问题

**双模式下车坐标系不一致**（待修复）：
- `collectSubLevelBlockPositions()` 读取的是 plot chunk 的固定世界坐标
- `getSubLevelPosition()` 返回 `logicalPose().position()` 的物理空间坐标
- 车辆物理运动后两者不同步
- 导致：手刹下车可能传送到车底，F 下车可能传送到百米高空
