## 第一阶段：车体局部坐标系瞄准

### 问题
炮塔瞄准的目标位置由摄像机射线提供（世界坐标），但是炮塔是安装在车体上的——车体俯仰/侧倾时，世界空间的水平方向不再对应车体局部水平方向，导致瞄准偏移。

### 方案：车体局部坐标系变换

在 `TurretAimController.drive()` 中增加方案 A（车体局部坐标系稳定瞄准）：

```
dir_world = normalize(target - gsPos)      // 世界空间目标方向
dir_local = q_veh⁻¹ · dir_world            // 变换到车体局部空间
localYaw  = atan2(dir_local.x, dir_local.z) // 局部偏航角
setMotor(-localYaw + offset)               // 取反：CCW+ → CW+
```

- 需要 `vehicleSubLevelId` 字段（在 `TurretBaseBlockEntity` 中新增，NBT 持久化）
- 回退方案 B：当无法获取车体 pose 时沿用旧世界空间算法

### 高低机同步改造

与方向机统一为**位置模式 PD 伺服**：
- `drivePitch()` 在砂轮局部空间计算绝对俯仰角：`atan2(-local.y, lh2)`
- `setTargetPitchAbsolute(degrees)` 直接设置
- 与方向机一样使用 position-mode setMotor

## 第二阶段：开火系统

### 架构设计

**核心原则**：瞄准与伤害分离，伤害从炮口出发。

```
每 client tick:
  ├─ performRaycast()  ← 摄像机位置 → TurretTargetC2SPacket（瞄准）
  └─ 鼠标左键 / 每 3 tick:
       └─ fireAllTurrets()  ← 遍历所有炮台
            ├─ 计算炮口位置 + 炮管朝向（避雷针位姿）
            ├─ raycastGeneric() ← 从炮口出发（方块→实体→SubLevel）
            ├─ 存储弹道数据（origin + hitPos）
            └─ WeaponFireC2SPacket（伤害）
                  └─ 服务端：invulnerableTime=0 → hurt()
```

### WeaponOverlay.java 重构

**之前**（单炮台 + 摄像机依赖）：
- `lastFireOrigin`/`lastFireHit`/`barrelFwd`：单组数据
- `setLastFireData(Vec3)`：用摄像机命中点算弹道终点
- `computeBarrelTipAndFwd()`：找到第一个炮台就 return

**之后**（多炮台 + 独立炮口）：
- `List<TurretFireInstance> activeFires`：多组弹道数据
- `fireAllTurrets(Minecraft mc)`：遍历所有炮台
- 每个炮台独立：计算位姿 → 炮口射线 → 存储 + 发包
- `raycastGeneric()`：通用射线检测（任意起点/方向）
- `getBlockFaceCenter()`：方块面几何中心

### 网络包

| 包 | 方向 | 职责 |
|---|:----:|------|
| `WeaponFireC2SPacket` | C→S | 命中坐标 (hitX, hitY, hitZ)，服务端找实体造成 2 点伤害 |
| `TurretTargetC2SPacket` | C→S | 瞄准目标坐标（每 tick 持续发送） |

`WeaponFireResultS2CPacket` 被移除——客户端本地计算弹道，服务端只做伤害。

## 第三阶段：细节打磨

### 1. 弹道渲染优化

**问题**：`BulletTrailRenderer` 每帧扫描所有 SubLevel Chunk 寻找炮塔，导致世界加载时严重卡顿。

**方案**：开火时在 `fireAllTurrets()` 中一次性计算炮口位置 + 炮管朝向 + 射线命中点，缓存到 `List<TurretFireInstance>`。渲染器直接读取，不做任何 SubLevel 扫描。

### 2. 弹道特效持续时间

- 初始 3 tick → 1 tick（太短不显示）→ 最终 **2 tick**
- 2 tick ≈ 0.1 秒，刚好能看到白色线条一闪而过

### 3. 方块面中心瞄准

`getBlockFaceCenter(BlockHitResult)` 根据 `hit.getDirection()` 返回命中面的几何中心：

| 面 | 坐标 |
|----|------|
| DOWN | (x+0.5, y, z+0.5) |
| UP | (x+0.5, y+1.0, z+0.5) |
| NORTH | (x+0.5, y+0.5, z) |
| SOUTH | (x+0.5, y+0.5, z+1.0) |
| WEST | (x, y+0.5, z+0.5) |
| EAST | (x+1.0, y+0.5, z+0.5) |

避免 `blockHit.getLocation()` 返回的精确表面点在方块底部或边缘导致的偏移。

### 4. 高低机 PD 增益

**问题**：避雷针是悬臂结构，重力臂长，PD 位置模式需稳态误差对抗重力矩。

**调整**（用户实测后自行调整到最优值）：
- `PITCH_SERVO_STIFFNESS`: 500 → 6000（×12，对抗重力下垂）
- `PITCH_SERVO_DAMPING`: 80 → 400（同比例 + 提高阻尼比防震荡）

### 5. 按住连发

`consumeClick()` → `isDown()`，按住鼠标左键以每 3 tick 一发持续射击，松开即停。使用 `mc.level.getGameTime()` 计时，避免长时间不操作后首次点击被错误拦截。

### 6. 无视无敌帧

在 `hurt()` 前加 `entity.invulnerableTime = 0` 清除受伤免疫，确保每发伤害都生效。

### 7. 调试轴线默认隐藏

`AxisLineRenderer.onRenderLevelStage()` 增加条件：
```java
if (!mc.getEntityRenderDispatcher().shouldRenderHitBoxes()) return;
```
默认隐藏，按 F3+B（显示碰撞箱）时显示。

### 8. 发射键改为鼠标左键

- 移除 B 键开火（`RAYCAST_FIRE_KEY` 从 `GLFW_KEY_B` 改为 `GLFW_MOUSE_BUTTON_LEFT`）
- 移除自动发射（每 5 tick 自动开火已删除）
- 移除 `InputEvent.MouseButton` 处理器（NeoForge 不允许监听抽象类）
- 改为在 `onClientTick` 中用 `isDown()` 检测

## 第四阶段：细节终调（06-11 终版）

### 9. 瞄准点精确命中坐标

**问题**：`getBlockFaceCenter()` 返回方块面的几何中心，但实际瞄准方向与该中心存在偏差，导致远距离瞄准误差明显。

**方案**：移除 `getBlockFaceCenter()`，直接使用 `blockHit.getLocation()` 射线与方块表面的精确交点。

**影响**：瞄准精度大幅提升，远距离射击不再有系统性的面中心偏移。

**变更**：
- `WeaponOverlay.java` 移除 `getBlockFaceCenter()` 方法及所有调用
- `performRaycast()` 和 `raycastGeneric()` 的 Javadoc 同步更新
- 不再依赖 `BlockHitResult.getBlockPos()` + `Direction` switch → 直接使用 `getLocation()`

### 10. 实体命中点 AABB 精确计算

**问题**：`entityHit.getLocation()` 在某些实现路径下可能返回 `entity.position()`（实体脚部坐标），而非射线与碰撞箱的实际交点，导致弹道白色线条终点落在实体脚部。

**方案**：手动调用 `entity.getBoundingBox().clip(origin, to)` 计算精确交点，兜底使用 `entityHit.getLocation()`。

```java
var bb = entityHit.getEntity().getBoundingBox();
var optHit = bb.clip(origin, to);
Vec3 preciseHit = optHit.orElse(entityHit.getLocation());
```

**影响**：
- ✅ 弹道白色线条终点指向实体碰撞箱上的精确命中点
- ✅ 伤害数据包 `WeaponFireC2SPacket` 发送正确的命中坐标
- ✅ 服务端 CRIT 粒子出现在正确位置

### 11. 射线穿透名单系统

**问题**：花草、树叶、农作物等方块不应阻挡瞄准/伤害射线——玩家直觉上子弹应穿过它们。

**方案**：双层穿透机制——
1. `ClipContext.Block.OUTLINE` → **`COLLIDER`**：碰撞箱为空的方块（花/草/农作物/藤曼/树苗/火把/红石线等）自然被射线忽略
2. **树叶穿透循环**：对有碰撞箱的树叶，用循环跳过并继续向后检测

```java
// COLLIDER 自然过滤无碰撞箱方块
BlockHitResult blockHit = mc.level.clip(
    new ClipContext(origin, to, ClipContext.Block.COLLIDER, ...)
);
// 树叶穿透循环（最多 32 层防无限）
for (int pass = 0; pass < 32 && blockHit.getType() != MISS; pass++) {
    if (!isPenetrable(hitState)) break;
    Vec3 advance = blockHit.getLocation().add(dir.scale(0.1));
    blockHit = mc.level.clip(new ClipContext(advance, to, ...));
}
```

`isPenetrable()` 覆盖范围：
- `BlockTags.LEAVES` — 所有树叶变种
- `Blocks.VINE` / `TWISTING_VINES` / `WEEPING_VINES` / `CAVE_VINES` — 所有藤曼

**影响**：
| 方块类型 | 之前 | 现在 |
|---------|:----:|:----:|
| 花/草/农作物/树苗 | 🟡 阻挡 | ✅ 穿透 |
| 树叶 | 🟡 阻挡 | ✅ 穿透 |
| 藤曼 | 🟡 阻挡 | ✅ 穿透 |
| 红石线/火把/按钮 | 🟡 阻挡 | ✅ 穿透 |
| 西瓜/南瓜/泥土 | 🟢 阻挡 | 🟢 阻挡 |
