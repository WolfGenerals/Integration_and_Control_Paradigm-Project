# 二十、显示值 vs 物理值分离

### 问题
摩擦需求比 `frictionDemandRatio` 使用户能感知抓地力利用程度。但若用物理层的 `frictionBudget`（含 `MIN_IMPULSE_MULTIPLIER=500` 膨胀），永远显示 17% 左右，无法反映真实打滑状态。

### 修复
```java
// 物理层（保持不变）：使用膨胀的 frictionBudget 保证操控响应
double frictionBudget = mu × max(adjustedSpringImpulse, nm × dt × 500);

// 显示层（改用自然预算）：不用 500× 放大器
double displayBasis = max(adjustedSpringImpulse, nm × dt × 20.0);
double displayBudget = mu × displayBasis;
this.frictionDemandRatio = totalDemand / max(displayBudget, 1.0);
```

自然预算 ≈ 2 倍重量冲量，显示值 > 100% 时直观反映"轮子在空转"。
