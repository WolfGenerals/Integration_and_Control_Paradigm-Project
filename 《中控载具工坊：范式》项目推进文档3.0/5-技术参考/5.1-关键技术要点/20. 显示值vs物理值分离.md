# 二十、显示值 vs 物理值分离

### 问题
摩擦需求比 `frictionDemandRatio` 使用户能感知抓地力利用程度。但若用物理层的 `frictionBudget`（含 `MIN_IMPULSE_MULTIPLIER` 膨胀），永远显示偏低，无法反映真实打滑状态。

### 演进历史

#### 阶段一：直接复用（原始方案）
```java
// 物理层：MIN_IMPULSE_MULTIPLIER = 500（膨胀 ~50×）
double frictionBudget = mu × max(springImpulse, nm × dt × 500);
// 显示层：直接复用 frictionBudget → 永远 ≈17%
```

#### 阶段二：物理与显示分离（06-08 中间状态）
```java
// 物理层保持不变：nm × dt × 500
double frictionBudget = mu × max(adjustedSpringImpulse, nm × dt × 500);
// 显示层改用自然预算（≈ 2× 重量冲量）
double displayBasis = max(adjustedSpringImpulse, nm × dt × 20.0);
```

#### 阶段三：MIN_IMPULSE 合理化（06-08 最终，当前方案）
```java
// 物理层 MULTIPLIER 从 500 降至 30（SuspensionConstants.MIN_IMPULSE_MULTIPLIER = 30）
// 物理预算与显示预算差距从 50× 缩小到 1.5×
double frictionBasis = max(adjustedSpringImpulse, nm × dt × 30);
double frictionBudget = mu × frictionBasis;

// 显示层仍用自然预算：显示值 >100% 直观反映抓地饱和
double displayBasis = max(adjustedSpringImpulse, nm × dt × 20.0);
double displayBudget = mu × displayBasis;
this.frictionDemandRatio = totalDemand / max(displayBudget, 1.0);
```

自然预算 ≈ 2 倍重量冲量，显示值 > 100% 时直观反映"轮子在空转"。
