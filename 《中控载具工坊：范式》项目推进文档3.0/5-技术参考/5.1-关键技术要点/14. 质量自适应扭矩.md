# 十四、质量自适应扭矩

### 核心公式

```java
double totalMass = ssl.getMassTracker().getMass();       // 总质量 (kg)
double effectiveTorque = totalMass * 9.81 * TORQUE_WEIGHT_RATIO;
this.effectiveTorque = Math.max(effectiveTorque, ENGINE_TORQUE * 0.5);
```

### 物理含义
- `TORQUE_WEIGHT_RATIO = 0.02`：每牛顿车重产生 0.02 Nm 扭矩
- 轻车（50 kg）：扭矩 ≈ 10 Nm
- 重车（12,000 kg）：扭矩 ≈ 2,354 Nm
- **功率/重量比一致**：所有车获得相同的推重比体验

### 与变速箱的配合
```java
// 变速箱用 effectiveTorque 替代旧的 ENGINE_TORQUE
totalWheelTorque = effectiveTorque × |ratio|    // ← 质量自适应
perWheelTorque = totalWheelTorque / numWheels
```

### ⚠ ENGINE_TORQUE 下限兜底

```java
this.effectiveTorque = Math.max(computedTorque, ENGINE_TORQUE * 0.5);
// ENGINE_TORQUE = 30.0 Nm → 下限 = 15.0 Nm
```

对于 70kg 标准测试车：`computedTorque = 70 × 9.81 × 0.02 = 13.73 Nm < 15.0 Nm`
→ **轻车的实际扭矩被下限兜底**，TORQUE_WEIGHT_RATIO 对轻车未完全生效。

当车重 > 77kg 时：`computedTorque > 15.0 Nm`，质量自适应开始线性增长。

### 安全性
- 服务端 `ServerSubLevel` 才有 `getMassTracker()`
- 异常捕获：若质量 API 调用失败，回退到 `ENGINE_TORQUE * 0.5`
